# Contributing
Thank you for your interest in contributing to this project!
Please follow these steps to help keep the project organized and maintain high quality.

1. Fork and clone the repository.
2. Create a new branch for your changes with a descriptive name.
3. Make your changes and commit them with descriptive messages.
4. Keep your branch up to date with the main repository.
5. Push your changes to your fork.
6. Submit a pull request to the main repository.
7. Be responsive to feedback and make necessary changes.
8. Follow the [Code of Conduct](CODE_OF_CONDUCT.md) for respectful communication.

## Choosing an issue to work on
- Read the Wiki to learn more about the project.
- Check the repo's issues to see if there is an issue that you would like to work on.
- If you would like to suggest a new feature, please open an issue first to discuss it with the maintainers.

## What to include in your pull request
- A clear description of the changes made, especially if it is related to an issue.
- Screenshots of the changes if applicable.
- Unit tests for any new code.
- Integration tests if applicable.
- Code that is correctly formatted.

## After your pull request is merged
- Update the Wiki with any new information or changes.
- Close the issue.
